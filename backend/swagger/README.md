# Swagger/OpenAPI Documentation

This directory contains the Swagger/OpenAPI documentation for the Project Task Management API.

## Files

- `swagger.yaml` - OpenAPI 3.0 specification file
- `index.html` - Swagger UI interface for viewing and testing the API
- `models/` - JSON schema models used by serverless-aws-documentation plugin

## Viewing Swagger Documentation

### Option 1: Local Swagger UI (Recommended for Development)

1. Install http-server (if not already installed):
   ```bash
   npm install -g http-server
   ```

2. Start the Swagger UI:
   ```bash
   cd backend
   npm run swagger
   ```

   This will open Swagger UI in your browser at `http://localhost:8080`

3. Or manually:
   ```bash
   cd backend/swagger
   npx http-server -p 8080 -o
   ```

### Option 2: Online Swagger Editor

1. Go to https://editor.swagger.io/
2. Click "File" → "Import file"
3. Upload or paste the contents of `swagger.yaml`
4. The API will be displayed with an interactive interface

### Option 3: After Deployment (AWS API Gateway)

After deploying to AWS:

1. Go to AWS Console → API Gateway
2. Select your API
3. Click on "Documentation" tab
4. The documentation will be automatically generated by the `serverless-aws-documentation` plugin

### Option 4: Import to Postman/Insomnia

1. **Postman**:
   - Open Postman
   - Click "Import"
   - Select "File" or "Link"
   - Upload `swagger.yaml` or paste the URL
   - All endpoints will be imported with examples

2. **Insomnia**:
   - Open Insomnia
   - Click "Create" → "Import/Export" → "Import Data"
   - Select "OpenAPI 3" → Upload `swagger.yaml`

## Testing the API

Once Swagger UI is running:

1. Make sure your local server is running: `npm run dev`
2. In Swagger UI, you can:
   - View all endpoints
   - See request/response schemas
   - Try out API calls directly
   - See example requests and responses

## Updating the Documentation

To update the documentation:

1. Edit `swagger.yaml` for the OpenAPI spec
2. The `models/` directory contains JSON schemas used by the serverless plugin
3. After changes, refresh Swagger UI to see updates

## Notes

- The local Swagger UI points to `http://localhost:3000/dev` by default
- Update the `servers` section in `swagger.yaml` if your API URL changes
- After deployment, update the server URL in `swagger.yaml` with your actual API Gateway URL
